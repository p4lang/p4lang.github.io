P4.org is proud to make available a wealth of software packages that enable you to get started on your data-plane programming journey. A good starting point is our rich set of tutorials. Our tutorials are developed as hands-on labs, and contain everything to get you started with experimenting in P4: an overview of the language, sets of exercises that increase in complexity, and a virtual machine pre-installed with all the software such that you can directly jump into data-plane programming. For the developer in you, we have made available all our software, so that you can start customizing it to your needs. This includes:
  • P4 code examples, including a data-center switch definition. It supports a large number of features, from L2/L3 processing, to Multicast, MPLS, ACL, SFLOW and even In-band Network Telemetry (INT).
  • The P4C compiler for both P414 and P416 language variants
    • . The compiler supports several backends: p4c-bmv2 that targets the behavioral simulator, p4c-ebpf that targets the Extended Berkeley Packet Framework, p4c-graphs that produces parser and table flow graph diagrams, and p4test -- a simple frontend verifier for P4 programs.
  • A simulation environment (BMv2) to run the P4 software switch, standalone or in Mininet. BMv2 is an extensible framework that currently supports a P414 pipeline (simple_switch). However it allows you to easily simulate additional externs by extending its modular system.
  • The specification and reference implementation for P4 Runtime, a runtime API and protocol for controlling data-plane programs.
  • A packet testing framework (PTF), written in Python, that enables configuring a switch and creating tests that send, receive, and verify packets of different formats. PTF interfaces with P4 Runtime.
  • P4app, an environment to tie it all together and deploy it using simple configuration rules.
And much more. Please feel free to browse around our github repositories. All our code is released under the Apache 2.0 license. As your interest grows, we encourage you to participate and contribute your code. Please join the P4 Developer mailing list to get updates, ask questions, and get answers. Note that you or your organization must be a member of P4.org in order for us to accept your contributions. Membership is free).
